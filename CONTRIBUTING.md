# Contributing to Awesome LLM Training & Inference

Thank you for your interest in contributing! This guide will help you make successful contributions.

## Quick Start

1. Find a missing framework, tool, or resource
2. Check [inclusion criteria](#inclusion-criteria)
3. Add to the appropriate category
4. Submit a pull request

## Inclusion Criteria

### Must Have

- ✅ Active development (last commit within 6 months)
- ✅ Clear documentation or README
- ✅ Minimum 10+ GitHub stars (core frameworks exempt)
- ✅ Relevance to LLM training or inference

### Nice to Have

- ⭐ 100+ stars for featured projects
- ⭐ Active community engagement
- ⭐ Recent releases or updates
- ⭐ Clear license

## Categories

### Training Frameworks
Deep learning frameworks and training libraries for LLM development.

### Inference Frameworks
High-performance inference engines and local deployment tools.

### Distributed Training
Tools for parallel and distributed LLM training.

### Model Compression & Quantization
Quantization tools and model compression frameworks.

### Data Processing
Data pipelines and processing tools for LLM training.

### Evaluation & Benchmarking
Evaluation frameworks and benchmarking tools.

### Deployment & Serving
Model serving platforms and deployment tools.

### Monitoring & Observability
Training and inference monitoring tools.

## Format Guidelines

### For Table Format

```markdown
| Framework | Stars | Description |
|-----------|-------|-------------|
| [name](url) | ![Stars](badge) | Description |
```

### For List Format

```markdown
- [name](url) - Description
```

## Pull Request Process

1. Fork the repository
2. Create a feature branch
3. Add your project following the format guidelines
4. Ensure all links are working
5. Submit a pull request
